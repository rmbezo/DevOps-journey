# Битовые сдвиги

"<<" - число увеличивается, Дописываем нули в конец.
* _Математически:_ Умножение на 2 за каждый шаг.
">>" - число уменьшается , Стираем цифры в конце.
* _Математически:_ Деление на 2 за каждый шаг.



### Как это выглядит в двоичном коде

Представим число **1**:

- `1 << 1` → `10` (в обычном мире это **2**)
- `1 << 2` → `100` (в обычном мире это **4**)
- `1 << 3` → `1000` (в обычном мире это **8**)


### Пример 

```
const Big = 1 << 100 
```
- **Что произошло:** Мы взяли `1` и приписали к ней **100 нулей**.
- **Почему Go не взорвался:** Это **константа**. Она живет в "идеальном мире" математики. В переменную `int` она не влезет, но внутри кода Go может производить с ней расчеты.


```
const Small = Big >> 99
```

- **Что произошло:** У огромного числа со 100 нулями стерли **99 нулей**.
- **Итог:** Остался `1` и один ноль (`10`). В десятичной системе это **2**.


Зачем это нужно?

1. **Скорость:** Процессор делает сдвиги быстрее, чем умножение.
2. **Флаги (Переключатели):** Храним много настроек "Да/Нет" внутри одного числа.
3. **Упаковка данных:** Когда нужно передать много информации по сети в очень сжатом виде.
